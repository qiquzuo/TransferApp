## 🚀 快速开始

### 方法一：便携版（推荐⭐⭐⭐⭐⭐）

**无需安装Python，开箱即用！**

## 📋 使用前准备

### 网络环境要求

✅ **必须条件：**
- 电脑和手机在同一WiFi网络或热点下
- 防火墙允许5000端口通信
- 浏览器支持现代Web标准（Chrome 60+、Firefox 55+、Safari 11+）

❌ **不支持：**
- 跨网络传输（如互联网）
- 不同子网的设备

---

### 检查网络连接

**Windows查看IP地址：**
```bash
ipconfig
```

查找 IPv4 地址，应该是 `192.168.x.x` 或 `10.x.x.x` 格式。

**确保手机和电脑IP在同一网段：**
- 电脑：192.168.1.100
- 手机：192.168.1.101 ✅ 可以连接
- 手机：192.168.2.100 ❌ 不在同一网段

## 🔧 配置说明

### 修改端口

编辑 `server.py` 最后一行：

```python
app.run(host='0.0.0.0', port=5000, debug=False)
# 改为其他端口，如：port=8080
```

重启服务器生效。

---

### 修改文件大小限制

编辑 `server.py`：

```python
MAX_FILE_SIZE = 100 * 1024 * 1024  # 100MB
# 改为其他大小，如：200 * 1024 * 1024 (200MB)
```

---

### 添加文件类型

编辑 `server.py`：

```python
ALLOWED_EXTENSIONS = {
    'png', 'jpg', 'jpeg', 'gif', 'bmp', 'webp', 'svg',

    'zip', 'rar', 'mp4', 'mp3', 'avi', 'mov',
    # 添加新类型，如：'psd', 'ai'
}
```

## ❓ 常见问题

### Q1: 手机无法连接？

**解决方法：**

1. **检查网络**
   - 确保手机和电脑在同一WiFi
   - 不要使用移动数据

2. **检查IP地址**
   ```bash
   # Windows
   ipconfig
   
   # 查找正确的IPv4地址（192.168.x.x）
   ```

3. **检查防火墙**
   - 打开"Windows Defender 防火墙"
   - 点击"高级设置"
   - 添加入站规则：TCP端口5000允许连接

4. **临时关闭杀毒软件测试**

---

### Q2: 上传文件失败？

**可能原因及解决：**

#### A. 文件过大

**限制：** 最大100MB

**解决：**
- 压缩文件
- 分割为大文件
- 修改 `MAX_FILE_SIZE` 配置

#### B. 文件格式不支持

**支持的格式：**
```
图片：png, jpg, jpeg, gif, bmp, webp, svg
文档：pdf, doc, docx, xls, xlsx, txt
压缩：zip, rar
媒体：mp4, mp3, avi, mov
```

**解决：** 转换文件格式或修改 `ALLOWED_EXTENSIONS`

#### C. 网络不稳定

**解决：**
- 使用有线网络
- 靠近路由器
- 重试上传

---

### Q3: 自动刷新不工作？

**诊断步骤：**

1. **打开开发者工具**
   ```
   按 F12 或 Ctrl+Shift+I
   ```

2. **查看Console**
   ```
   应该看到：✅ 自动刷新已启动
   如果有错误，记录错误信息
   ```

3. **查看Network面板**
   ```
   过滤：history
   应该每0.1秒看到一个 GET /api/history 请求
   状态码应该是 200
   ```

**常见问题解决：**

- **JavaScript错误：** 清除浏览器缓存（Ctrl+F5）
- **API请求失败：** 检查服务器是否正常运行
- **定时器被清除：** 刷新页面重新启动

---

### Q4: 图片无法预览？

**可能原因：**

#### A. 文件不存在

**检查：**
```bash
# 查看 received_files 文件夹
dir received_files
```

**解决：** 重新上传文件

#### B. 浏览器兼容性

**解决：** 在其他浏览器中打开

---

### Q5: 如何清空历史记录？

**方法1：** 点击"清空全部"按钮  
**方法2：** 删除 `received_files/` 文件夹中的所有文件  
**方法3：** 重启服务器（内存中的记录会清空）

---

### Q6: 下载文件格式错误？

**原因：** 浏览器默认行为

**解决：**
- 右键链接 → "另存为"
- 手动添加正确的扩展名

---

### Q7: 页面显示异常？

**可能原因：**

#### A. 浏览器缓存

**解决：**
```
硬刷新：Ctrl+F5
或清除缓存
```

#### B. 浏览器版本过旧

**要求：**
- Chrome 60+
- Firefox 55+
- Safari 11+
- Edge 79+

**解决：** 更新浏览器

## 🔍 故障排查

### 查看服务器日志

**控制台输出示例：**
```
192.168.0.102 - - [27/Apr/2026 19:31:47] "GET /api/history HTTP/1.1" 200 -
```

**正常日志：**
- 状态码 200
- 请求路径正确

**异常日志：**
- 状态码 404/500
- 错误信息

---

### 浏览器Console调试

**常用命令：**

```javascript
// 检查API是否正常
fetch('/api/history').then(r => r.json()).then(console.log)

// 检查内存使用
console.memory

// 查看当前URL
window.location.href

// 检查User-Agent（设备检测）
navigator.userAgent
```

---

### Network面板分析

**检查要点：**

1. **请求频率：** 应该每0.1秒一个 `/api/history`
2. **响应时间：** 应该 <200ms
3. **状态码：** 应该都是 200
4. **响应大小：** 应该 <5KB

## 🛠️ 快速修复脚本

### 重置服务器

创建 `reset.bat`：

```batch
@echo off
echo 正在重置服务器...

:: 停止服务器
taskkill /F /IM python.exe

:: 清理文件
del received_files\*.* /Q

:: 重新启动
start python server.py

echo 服务器已重置！
pause
```

---
