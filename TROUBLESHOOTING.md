# 🔧 故障排查指南

> 常见问题及解决方案

---

## 🚨 快速诊断

### 问题检查清单

遇到问题时，按顺序检查：

1. ✅ 服务器是否启动？
2. ✅ 网络连接是否正常？
3. ✅ IP地址是否正确？
4. ✅ 防火墙是否阻止？
5. ✅ 浏览器是否有错误？

---

## 📋 常见问题

### 1️⃣ 服务器无法启动

**症状：**
```
python: command not found
或
ModuleNotFoundError: No module named 'flask'
```

**原因：**
- Python未安装
- 依赖未安装

**解决方法：**

**步骤1：检查Python**
```bash
python --version
# 应显示：Python 3.8.x 或更高
```

**步骤2：安装依赖**
```bash
pip install -r requirements.txt
```

**步骤3：重新启动**
```bash
python server.py
```

---

### 2️⃣ 手机无法连接

**症状：**
- 浏览器显示"无法访问此网站"
- 连接超时

**可能原因：**

#### A. 不在同一网络

**检查：**
- 手机和电脑是否连接同一WiFi？
- 是否使用移动数据？

**解决：**
确保两者在同一局域网

---

#### B. IP地址错误

**检查：**
```bash
# Windows
ipconfig

# 查找 IPv4 地址
# 应该是 192.168.x.x 或 10.x.x.x
```

**解决：**
使用正确的局域网IP，不是 `127.0.0.1`

---

#### C. 防火墙阻止

**Windows防火墙设置：**

1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 点击"入站规则" → "新建规则"
4. 选择"端口" → "TCP" → "特定本地端口: 5000"
5. 选择"允许连接"
6. 完成

**或临时关闭防火墙测试**

---

#### D. 杀毒软件阻止

**解决：**
- 临时关闭杀毒软件
- 或将 `server.py` 添加到白名单

---

### 3️⃣ 上传文件失败

**症状：**
- 显示"上传失败"
- 进度条卡住

**可能原因：**

#### A. 文件过大

**限制：** 最大100MB

**解决：**
- 压缩文件
- 分割为大文件
- 修改 `server.py` 中的 `MAX_FILE_SIZE`

---

#### B. 文件格式不支持

**支持的格式：**
```
图片：png, jpg, jpeg, gif, bmp, webp, svg
文档：pdf, doc, docx, xls, xlsx, txt
压缩：zip, rar
媒体：mp4, mp3, avi, mov
```

**解决：**
转换文件格式或修改 `ALLOWED_EXTENSIONS`

---

#### C. 网络不稳定

**解决：**
- 使用有线网络
- 靠近路由器
- 重试上传

---

### 4️⃣ 自动刷新不工作

**症状：**
- 历史记录不更新
- 需要手动刷新页面

**诊断步骤：**

**步骤1：打开开发者工具**
```
按 F12 或 Ctrl+Shift+I
```

**步骤2：查看Console**
```
应该看到：✅ 自动刷新已启动（每2秒）
如果有错误，记录错误信息
```

**步骤3：查看Network**
```
过滤：history
应该每2秒看到一个 GET /api/history 请求
状态码应该是 200
```

**常见问题：**

#### A. JavaScript错误

**解决：**
- 清除浏览器缓存（Ctrl+F5）
- 尝试其他浏览器

---

#### B. API请求失败

**检查：**
```javascript
// 在Console执行
fetch('/api/history').then(r => r.json()).then(console.log)
```

**应该返回：**
```json
{success: true, history: [], count: 0}
```

---

#### C. 定时器被清除

**解决：**
刷新页面重新启动

---

### 5️⃣ 图片无法预览

**症状：**
- 点击图片无反应
- 显示 broken image 图标

**可能原因：**

#### A. 文件不存在

**检查：**
```bash
# 查看 received_files 文件夹
dir received_files
```

**解决：**
重新上传文件

---

#### B. MIME类型错误

**解决：**
服务器会自动检测，通常无需手动处理

---

#### C. 浏览器兼容性

**测试：**
在其他浏览器中打开

---

### 6️⃣ 下载文件格式错误

**症状：**
- 下载的文件无法打开
- 文件扩展名不对

**原因：**
浏览器默认行为

**解决：**
- 右键链接 → "另存为"
- 手动添加正确的扩展名

---

### 7️⃣ 页面显示异常

**症状：**
- 布局错乱
- 样式丢失

**可能原因：**

#### A. 浏览器缓存

**解决：**
```
硬刷新：Ctrl+F5
或清除缓存
```

---

#### B. 浏览器版本过旧

**要求：**
- Chrome 60+
- Firefox 55+
- Safari 11+
- Edge 79+

**解决：**
更新浏览器

---

#### C. CSS未加载

**检查：**
```
F12 → Network
查看是否有CSS文件加载失败
```

---

### 8️⃣ 内存占用过高

**症状：**
- 电脑变慢
- 浏览器卡顿

**可能原因：**

#### A. 历史记录过多

**解决：**
- 清空历史记录
- 重启服务器

---

#### B. 大文件过多

**解决：**
```bash
# 清理 received_files 文件夹
del received_files\*.*
```

---

#### C. 内存泄漏

**检查：**
```javascript
// Console执行
console.memory.usedJSHeapSize / 1024 / 1024 + ' MB'
```

**正常值：** <30MB  
**如果持续增长：** 刷新页面

---

### 9️⃣ 二维码无法扫描

**症状：**
- 扫码无反应
- 识别失败

**可能原因：**

#### A. 二维码太小

**解决：**
- 放大浏览器窗口
- 调整屏幕亮度

---

#### B. 光线不好

**解决：**
增加环境光线

---

#### C. 手动输入

**备选方案：**
直接输入显示的URL地址

---

### 🔟 文本复制失败

**症状：**
- 点击复制按钮无反应
- 剪贴板无内容

**可能原因：**

#### A. 浏览器权限

**解决：**
- 允许网站访问剪贴板
- 或使用手动复制（Ctrl+C）

---

#### B. HTTPS要求

**说明：**
某些浏览器要求HTTPS才能访问剪贴板

**解决：**
使用手动复制

---

## 🔍 高级诊断

### 查看服务器日志

**控制台输出：**
```
192.168.0.102 - - [27/Apr/2026 19:31:47] "GET /api/history HTTP/1.1" 200 -
```

**正常日志：**
- 状态码 200
- 请求路径正确

**异常日志：**
- 状态码 404/500
- 错误信息

---

### 浏览器Console调试

**常用命令：**

```javascript
// 检查API是否正常
fetch('/api/history').then(r => r.json()).then(console.log)

// 检查内存使用
console.memory

// 查看当前URL
window.location.href

// 检查User-Agent（设备检测）
navigator.userAgent
```

---

### Network面板分析

**检查要点：**

1. **请求频率：** 应该每2秒一个 `/api/history`
2. **响应时间：** 应该 <200ms
3. **状态码：** 应该都是 200
4. **响应大小：** 应该 <5KB

## 🎯 快速修复脚本

### 重置服务器

创建 `reset.bat`：

```batch
@echo off
echo 正在重置服务器...

:: 停止服务器
taskkill /F /IM python.exe

:: 清理文件
del received_files\*.* /Q

:: 重新启动
start python server.py

echo 服务器已重置！
pause
```

---

### 清除浏览器缓存

**Chrome/Edge：**
```
Ctrl+Shift+Delete
选择"缓存的图片和文件"
点击"清除数据"
```

**Firefox：**
```
Ctrl+Shift+Delete
选择"缓存"
点击"立即清除"
```
